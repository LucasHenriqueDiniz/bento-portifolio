Discord Webhook	DISCORD_WEBHOOK_URL	POST {DISCORD_WEBHOOK_URL}	Discord não recomenda hardcode de limite; webhooks seguem rate limits por rota via headers, e os limites podem variar.	O endpoint aceita payload JSON como { content?, username?, avatar_url?, embeds?, allowed_mentions?, components?, attachments? }. Respostas podem ser 2xx, 429, etc.; o comportamento de limite vem nos headers X-RateLimit-*.

Notas curtas

Não incluí rotas internas do app, só APIs externas.
Em Last.fm, Steam, Lyfta e Lanyard, onde a documentação pública não expõe um limite numérico claro, marquei isso explicitamente.
No caso do Jikan, o envelope data é o padrão da API v4; os campos exatos variam por endpoint.
Discord Webhook e Turnstile são integrações HTTP, mesmo não sendo “APIs de dados” no mesmo estilo de Last.fm/MAL.
